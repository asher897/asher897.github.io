Cross-Attention for Domain Transfer in Medical Image Segmentation

For my Honours research at Wits I built a segmentation model that attends to a bag of context images while processing each input — forcing it to learn what's general about a disease, not just what's specific to a training image. The results were mixed in the best possible way: the mechanism worked, the segmentation didn't improve, and that distinction turned out to be the most interesting finding.

The Problem: Models That Don't Travel

Medical image segmentation has a domain transfer problem. A model trained to segment COVID-19 lesions in chest X-rays can perform well on its training distribution and then fail quietly on a different dataset of the same disease — different hospital, different scanner, slightly different patient population. The deeper problem is that it will often fail even more sharply on a related disease it was never trained on at all.

This matters practically because annotating medical images is expensive and slow — every pixel label requires a radiologist. The result is that most medical segmentation datasets are small by computer vision standards, models overfit to their source domain, and generalisation suffers. The question I wanted to explore was: can we give the model a structural mechanism that forces it to learn features general to a disease category, rather than features specific to a training image?

A real instance of this problem exists in South African radiology: CAD systems trained to detect tuberculosis in chest X-rays degrade significantly when silicotuberculosis cases are introduced. The visual presentations are similar enough to share features, but different enough to break models that learned too specifically. That's the class of problem this research aimed to address.

The Idea: Bags and Attention

The core proposal was to process images not in isolation but in bags — a focus image (the one being segmented) accompanied by a set of context images, all sharing the same disease label. The model would attend to the context images' features while processing the focus image, using them as a reference signal for what the disease looks like across different patients and presentations.

The intuition draws a parallel with few-shot learning: if you show a model not just the image you want segmented but also several other examples of the same condition, it has more information to anchor what it's looking for. The difference from few-shot learning is that the context images here might come from a different category to the focus image — which is what creates a training signal. The model must learn to classify whether the bag is same-category or mixed, and in doing so, must learn to use the contextual features meaningfully.

The training signal: by asking the model to classify whether the focus image and context images share a label, we force it to actually use the context features. A model that ignores context images cannot solve the classification task. This is the mechanism that drives genuine feature learning rather than incidental correlation.

Architecture: U-Net with CCT and CCA Modules

The base architecture is a U-Net with a ResNet50 encoder pre-trained on ImageNet weights. This was chosen over more complex transformer-based architectures — TransUNet, Swin-UNet — partly because ViT-based models are known to underperform CNNs on small datasets, and partly because the medical datasets available here were relatively modest in size.

The cross-attention mechanism was built by adapting the CCT (Channel-wise Cross Fusion Transformer) and CCA (Channel-wise Cross Attention) modules from UCTransNet, with a key modification: rather than attending within a single image's feature maps, the CCT was reworked to attend between the focus image and the bag of context images.

The CCT module in detail

The critical modification to UCTransNet's CCT is in how Q, K, and V are formed. In the original, all embeddings come from the same image's encoder features across skip connections — it is self-attention. In this research, the queries come from the context images and the keys and values come from the focus image:

CCT — cross-image attention setup 
# Focus image feature map at skip level i: Fi  — shape (N, C, H, W)
# Bag of context feature maps: BCi            — shape (N, B, C, H, W)
# where B = bag size (4 context images)

# Both are layer-normalised, then patched via average pooling
# Patch size PS = H / P  (P is a per-skip-level hyperparameter)

# Context images → queries (what to look for)
# Focus image    → keys and values (where and what to retrieve)
Qi = depthwise_conv(avg_pool(BCi), W_Q)   # (N, B·C, H_p·W_p)
Ki = depthwise_conv(avg_pool(Fi),  W_K)   # (N, C,   H_p·W_p)
Vi = depthwise_conv(avg_pool(Fi),  W_V)   # (N, C,   H_p·W_p)

# Multi-head attention along the channel dimension
MHCAi = multihead_attention(Q=Qi, K=Ki, V=Vi)

# Upsample back to focus image spatial dims, then residual connection
Oi = layer_norm(Fi + depthwise_conv(upsample(MHCAi)))
# Oi: (N, C, H, W) — same shape as Fi, enriched with context

The channel-wise patching is worth explaining: rather than attending over every spatial position (which would be prohibitively expensive for image-sized tensors), the feature maps are divided into patches and average-pooled — each token in the attention sequence represents a patch's channel-wise summary. The patch number hyperparameter varies per skip level (8, 8, 16, 16, 32 from deepest to shallowest) to match the spatial scale at each level.

The CCA module

The CCT output Oi is then passed through the CCA before being concatenated onto the decoder. The CCA uses global average pooling to compute a channel-wise gating signal — combining information from Oi and the decoder's previous layer output — that modulates how much of the attended features actually flows into the decoder:

CCA — channel-wise gating 
# Oi: CCT output — attended focus features
# Di: decoder output at this level (post-upsample)

# Global average pool each to C×1×1
Mi = relu(L_O @ gap(Oi)) + relu(L_D @ gap(Di))

# Sigmoid gate: how much of the CCT output is useful here?
Ai  = sigmoid(Mi)            # (N, C, 1, 1)
Obi = Ai * Oi                # element-wise channel scaling

# Concatenate gated CCT output onto decoder features (as in standard U-Net)
decoder_input = concat([Obi, Di], dim=channel)

The classification head and loss function

A binary classification head sits on top of the segmentation output. It flattens the segmentation map and passes it through a linear layer to predict whether the focus image shares its label with the context images (1) or not (0). The full loss function combines three terms:

composite loss function 
Loss = λ_seg · (λ_bce · BCE(seg_pred, seg_gt)
             + λ_iou · (1 - IoU(seg_pred, seg_gt)))
     + λ_ce  · CE(class_pred, bag_label)

# Hyperparameters used:
# λ_seg = 1.0,  λ_bce = 1.0,  λ_iou = 0.5,  λ_ce = 0.4
# Bag label probability (same-category bags): 0.65
# Bag size: 4 context images per focus image

Training pipeline

Training proceeded in two stages. First, the U-Net with the ResNet50 backbone was pre-trained on the COVID-QU-Ex dataset using only focus images — no bagging, no classification head — until early stopping on validation loss. Second, the pre-trained weights were loaded and the CCT, CCA, and classification head were added and trained jointly. Freezing the encoder was attempted but produced worse results than training the full model end-to-end.

Datasets and Evaluation Setup

All experiments used chest X-ray datasets. The model was trained on the COVID-QU-Ex dataset — a curated collection of COVID-19, Pneumonia, and Non-infection CXRs, with segmentation masks provided for COVID-19 lesions (pneumonia masks are all-zero, as pneumonia was not the target label). Domain transfer was evaluated on the Mendeley COVID-19 Segmentation and Severity Scoring dataset, which was never seen during training.

The evaluation dataset served two purposes: it measured how well the segmentation generalised to unseen data with related pathology, and it was used to evaluate the bagging classifier's ability to correctly identify bag category on data it hadn't been trained on.

Two things stand out. First, the ResNet50 backbone provides a significant boost over the plain U-Net — 0.8157 to 0.8507 Dice. Second, and interestingly, the attention gate in the Attention U-Net provides a meaningful improvement on the base U-Net (0.8157 → 0.8210) but adds nothing when the ResNet50 backbone is already present (0.8507 → 0.8457 — actually slightly worse). The pre-trained backbone is already doing the feature enrichment that the attention gate was helping with. The plain U-Net + ResNet50 was selected as the segmentation backbone going forward.

The bagging model did not improve segmentation performance on either dataset. On the training test subset, performance dropped noticeably. On the unseen evaluation dataset, the drop was smaller but still present. These are the results as they are.

Why this is still a meaningful result: the datasets used — COVID-19 chest X-ray segmentation — do not present the kind of difficulty that bagging was designed to solve. A plain U-Net with a strong backbone can already segment COVID-19 lesions well. The hypothesis was specifically that bagging would help when there is a genuine difficulty in extracting disease-specific features. Testing it on an already-solved problem means the comparison baseline is already near its ceiling.

Bagging classification — the part that worked

The more revealing result is the classification performance. The model was asked to predict whether the focus image shared its label with the context images — a task that can only be solved by genuinely using the contextual features:

DatasetPrecisionRecallF1
COVID-QU-Ex (training)0.950.890.91
Mendeley (unseen evaluation)0.730.720.72

On the training dataset, the model achieves strong classification performance — an F1 of 0.91, correctly identifying same-category bags with 95% precision. On the unseen evaluation dataset, performance drops to an F1 of 0.72, but this is still a meaningful result: the model has never seen data from this dataset, yet it can generalise its understanding of what makes a bag same-category or different-category across datasets.

The validation CE loss curve confirmed the mechanism was learning during training — the classification loss decreased steadily across training steps, demonstrating that the network was actively learning to use the contextual features rather than ignoring them.

COVID-19 classification — loss tells the story

Both variants — with and without bagging — achieved comparable classification scores on the evaluation dataset (Precision, Recall, F1 all 0.82 for both). But the loss diverged meaningfully: the standard U-Net produced a loss of 4.38 versus 1.39 for the bagging variant. The bagging model's predictions were substantially more confident, even when the hard classification metrics were the same. In a clinical context, prediction confidence matters — a model that arrives at the right answer but with weak confidence is less useful than one that is reliably certain.

What This Means

The research demonstrates a mechanism — not a solved problem. The model can learn to use cross-image contextual features to identify the categorical context of the data it's processing. That's the key finding. It doesn't yet translate into better segmentation maps on the datasets tested, but those datasets weren't the hard case.

The paper's future work section points at the right application: the silicotuberculosis problem in South African radiology, where CAD systems trained on TB data degrade when silicoTB cases are introduced. That's a setting where the standard model genuinely struggles, and where attending to a bag of same-category context images could provide the anchor the model needs. Testing the approach there, or on any dataset where there is a documented and specific difficulty in feature extraction, would be the meaningful next experiment.

On the segmentation performance drop: it's worth noting that the bagging model carries a harder training objective — it is simultaneously learning to segment and to classify bag membership. The segmentation performance drop on the training dataset may partly reflect this added optimisation complexity. With more training epochs or a staged fine-tuning approach (train classification to convergence first, then unfreeze segmentation), the gap might narrow.

Key Takeaways

1. The mechanism generalises. The bagging classifier achieving F1 of 0.72 on an entirely unseen dataset — with no target-domain labels — is the strongest evidence that the model is learning something general about disease categories, not just memorising training data patterns.

2. Evaluate on hard problems. Testing a domain transfer technique on a dataset where the baseline already achieves 0.85 Dice tells you little. The hypothesis needs to be tested where domain transfer genuinely fails — the silicoTB case, or any setting where standard models show a documented performance cliff.

3. The CCT query/key inversion is the key design choice. Putting the context images in the query and the focus image in the key/value positions is what forces the model to ask "given what I know about this disease from other examples, what should I be looking for in this image?" — rather than self-attending within the focus image.

4. Loss confidence can matter as much as accuracy. In binary classification tasks where both models achieve the same hard metrics, the loss divergence (4.38 vs 1.39) is meaningful signal — especially in clinical applications where calibrated confidence matters for downstream decision-making.

5. Transformer modules on small medical datasets need a strong CNN foundation. The attention gate added nothing on top of the ResNet50 backbone, and the ViT-based literature shows the same pattern. In low-data regimes, a well-initialised CNN encoder is hard to beat with attention alone — the two need to be combined thoughtfully.
