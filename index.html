Recovery Pathway Modelling — Arthroplasty Outcomes
Professional Work
Client project — code proprietary

Applied ML to identify distinct post-operative recovery trajectories for knee and hip arthroplasty patients, surfacing the clinical features most predictive of different recovery experiences.

scikit-learn Polars pandas SHAP Airflow
Problem
We wanted to understand why patients undergoing knee and hip arthroplasty experienced such varied recovery outcomes — some recovering quickly, others facing prolonged rehabilitation or complications — and what factors drove those differences.
Approach
Designed and built end-to-end data pipelines in Polars to clean and structure longitudinal patient episode data. Applied clinical-based clustering to identify distinct recovery pathway groups, then used classifier models with SHAP-based feature importance and statistical analysis to characterise what differentiated each cohort — across clinical and procedural.
Methods
Clinical clustering for pathway segmentation; Logistic regression and gradient boosted trees (XGBoost, LightGBM) for classifying recovery; SHAP for post-hoc explainability; Polars for high-performance in-memory feature engineering.
Outcome
Delivered a clear taxonomy of recovery profiles with interpretable feature drivers, enabling actuarial teams to model risk more precisely and informing clinical recommendations around patient selection and rehabilitation planning.
Async Microservice Worker Framework
Professional Work
Client project — code proprietary

Designed and built an asynchronous Python worker framework that eliminated synchronous bottlenecks across a bank's microservice infrastructure, significantly improving throughput and reducing end-to-end latency.

Python asyncio RabbitMQ Redis Docker Kubernetes
Problem
A major bank's microservice infrastructure was suffering from significant latency caused by synchronous worker processes — each task blocked its thread until completion, creating throughput ceilings that became critical bottlenecks under real-world load.
Approach
Independently diagnosed the root cause through production metrics analysis, then designed an async worker framework from scratch. The framework abstracted concurrency management away from individual services, allowing teams to adopt async execution through configuration rather than code rewrites. Simultaneously redesigned the RabbitMQ integration layer that the workers depended on from a polling-based approach to a notification-based approach.
Methods
Python asyncio for non-blocking execution; redesigned RabbitMQ consumer/producer patterns to eliminate unnecessary round-trips; Redis for distributed state; containerised with Docker and deployed on Kubernetes (AKS & OpenShift).
Outcome
~50% reduction in end-to-end latency across multiple process flows; CPU usage halved on numerous services. The async framework was adopted across the broader microservice infrastructure and became the standard pattern for new worker development.
Transfer Learning & Seeded Iterated Learning on AfriBERTa
University Project
Group project · University of the Witwatersrand · 2023

Investigated transfer learning and language drift in multilingual NLP using AfriBERTa — a transformer model trained on 11 low-resource African languages. Fine-tuned the model on cross-lingual news topic classification tasks across Hausa and Swahili, then implemented a Seeded Iterated Learning (SIL) algorithm to combat language drift: the tendency of fine-tuned models to lose syntactic and semantic properties outside their target task.

Python PyTorch HuggingFace Transformers AfriBERTa NLP Transfer Learning
Problem
When fine-tuning language models sequentially on multiple downstream tasks, models undergo language drift — gradually losing the syntactic and semantic properties of language learned during pre-training, becoming increasingly specialised at the expense of general linguistic understanding.
Approach
Implemented a teacher-student SIL framework on top of AfriBERTa. The teacher agent was fine-tuned on the target task using real labels, then used to generate soft label distributions for the student agent to train on during the imitation phase. This indirect training signal preserves broader linguistic knowledge rather than overfitting to task-specific labels. Experimented with imitation phase lengths of 1, 3, and 5 epochs to assess their effect on drift mitigation.
Methods
AfriBERTa fine-tuned on MasakhaNEWS dataset across Hausa and Swahili news topic classification; AdamW optimiser with cross-entropy loss; softmax output distributions as soft labels; label frequency distribution analysis to visualise and quantify language drift across tasks.
Outcome
SIL successfully combated language drift — the SIL model maintained a consistent label distribution across both tasks, whereas the baseline model showed pronounced distributional shift indicative of drift. The trade-off was a modest reduction in raw accuracy and F1 metrics (0.76 vs 0.81 on task 1), an expected consequence of training on soft rather than hard labels.
Deep Q-Network Agent — Atari Pong
University Project
Group project · University of the Witwatersrand · 2023

Implemented the Nature DQN architecture from scratch to train a reinforcement learning agent on Atari Pong (PongNoFrameskip-v4). Built the full training pipeline including experience replay, epsilon-greedy exploration, and periodic target network updates.

Python PyTorch OpenAI Gym Reinforcement Learning Deep Q-Network
Problem
Train an agent to play Atari Pong directly from raw pixel input using reinforcement learning — learning an optimal policy purely from environment interactions and reward signals, with no hand-crafted features or game-specific logic.
Approach
Implemented the Nature DQN architecture: a three-layer convolutional network to extract spatial features from stacked 84×84 greyscale frames, followed by fully connected layers mapping to Q-values over the action space. Trained using experience replay to decorrelate samples, a frozen target network updated periodically to stabilise learning targets, and an epsilon-greedy policy with linear decay to balance exploration and exploitation.
Methods
CNN architecture: Conv(8×8, s=4) → Conv(4×4, s=2) → Conv(3×3, s=1) → FC(512) → Q-values; MSE TD-error loss; Adam optimiser (lr=1e-4); replay buffer of 50k transitions; target network synced every 100 steps.
Outcome
Agent learned a competitive Pong policy from pixel observations over 1M environment steps, with tracked episode returns and mean Q-value trends confirming stable convergence.
